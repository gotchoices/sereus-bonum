# Spec: Web Internationalization

**See also:** [Shared i18n spec](../../i18n.md) for principles and dictionary format.

## Web-Specific Details

The web app uses Svelte stores for reactive language switching.

## Usage

All user-facing text uses the translation function:

```svelte
<script>
  import { t } from '$lib/i18n';
</script>

<button>{$t('common.save')}</button>
<h1>{$t('accounts.title')}</h1>
<p>{$t('balance_sheet.as_of', { date: '2024-12-12' })}</p>
```

The `$` prefix makes translations reactive - if the user changes language in Settings, all text updates automatically.

## Dictionary Organization

The English dictionary is organized by functional area:

- `app.*` - Application name, tagline
- `nav.*` - Navigation labels
- `common.*` - Buttons, actions used everywhere (save, cancel, delete, etc.)
- `entities.*` - Entity management screen
- `accounts.*` - Accounts view
- `ledger.*` - Transaction entry
- `search.*` - Transaction search
- `settings.*` - Settings screen
- `account_types.*` - Account type names (Assets, Liabilities, etc.)
- `errors.*` - Error messages

## MVP Scope

**Language:** English only

**All text externalized:** No hard-coded strings in components. Every piece of UI text comes from the dictionary, including:
- Button labels
- Screen titles
- Error messages
- Placeholder text
- Help text
- Confirmation dialogs

**Infrastructure ready:** Code structure supports adding new languages by simply adding new dictionary files (e.g., `es.json` for Spanish).

## Future Enhancements

**Additional Languages:**
- Spanish, French, German initially

- Community translation platform

**Advanced Features:**
- Pluralization rules (1 item vs 2 items)
